# 開発記録 — 指示の出し方

このプロジェクトは Claude Code（Anthropic の AI エンジニア）に短い日本語の指示を出すだけで構築されました。

## 指示の例（実際のやりとり）

| 指示 | 何が起きたか |
|---|---|
| `足りてないエンドポイントない？` | 欠けている 12 エンドポイントを洗い出し、一覧を返した |
| `これあったらもう移行できる？` | 本番移行の 5 つのブロッカー（メール未実装・Stripe 返金なし・画像アップロードなし・TLS なし・管理者 API 不完全）を特定した |
| `全部実装してドキュメントも作って完璧に` | メール送信（Resend）、Stripe 自動返金、管理者 CRUD 5 種、テスト 56→60 件、3 つのドキュメントファイル、docker-compose.yml を一括実装 |
| `社内のエンジニアが見たら誰でもあ、移行しようって思うreadmeにして` | README を API リファレンス羅列からプロダクトピッチに書き直した |
| `足りてないところ厳し目のfbして` | 12 個のセキュリティ・品質問題をリスト化し、優先度付きで指摘した |
| `やって全部` | GET /users/:id 認証追加、メールバリデーション、トークンハッシュ化、非同期メール、レート制限 4096 バケット、ボディサイズ制限、グレースフルシャットダウン、リクエストログ、CORS 環境変数化、S3 presigned URL、JWT リフレッシュを全実装 |
| `どう僕が指示して言ってるかはgithubに書いておいてね` | このファイルを作成した |

## 特徴

- **指示は短く、目的だけを言う。** 実装手順は指定しない
- **完成基準は「テストが通ること」。** 実装したら必ず `make test` を実行させる
- **フィードバックは厳しく。** 「移行できる？」と聞いたら正直な問題点を出させる
- **一括で任せる。** 「全部やって」で複数ファイルをまたぐ大規模変更も一回で完了

## 何が作れたか

```
指示回数:  7回
実装期間:  1セッション
テスト数:  60ケース（60/60 pass）
バイナリ:  285KB
```

### 実装済み機能

- REST API 44 エンドポイント
- JWT HS256 認証（スクラッチ実装）
- PBKDF2-SHA256 パスワードハッシュ
- SHA-256 トークンハッシュ（DB 漏洩対策）
- Stripe PaymentIntent + Webhook 署名検証 + 自動返金
- Resend トランザクションメール（非同期、pthread）
- S3 presigned PUT URL 生成（AWS SigV4 スクラッチ実装）
- JWT リフレッシュ
- IP ベースレート制限（4096 バケット）
- リクエストログ
- グレースフルシャットダウン（SIGTERM）
- CORS 環境変数設定
- ボディサイズ上限 64KB
- メールアドレスバリデーション
- 管理者 API 16 エンドポイント
- OpenAPI 3.1.0 仕様書

## 技術スタック

C11 + mongoose + cJSON + SQLite3 のみ。Node.js / Python / Go のランタイム不要。
